# 一. 集群概念
**集群(Cluster):**

```shell
计算机集群简称集群，是一种计算机系统，它通过一组松散集成的计算机软件(和/或)硬件连接起来高度紧密地协作完成计算工作。在某种意义上，
他们可以被看作是一台计算机。集群系统中的单个计算机通常称为节点，通常通过局域网连接，但也有其它的可能连接方式。集群计算机通常用
来改进单个计算机的计算速度(和/或)可靠性。一般情况下集群计算机比单个计算机，工作站或超级计算机性能价格比要高得多。
```
- 并发量：Redis理论10W/秒的速度，但是10+的用户并发或离线计算缓冲如何设计？
- 数据量：机器(16~256G内存) `数据过大`或网络流量超越了`千兆网卡`等

集群主要解决了单机网络、容量、CPU的瓶颈问题，也就是解决了水平扩展的问题。

# 二. 数据分布

- `顺序分区`和`哈希分区`两种分区方式

| 分布方式 | 特点                                         | 典型产品                             |
| -------- | -------------------------------------------- | ------------------------------------ |
| 哈希分布 | 离散度高<br/>无法顺序访问<br/>支持批量操作   | 一致性哈希Memcache<br/>Redis Cluster |
| 顺序分布 | 数据容易倾斜<br/>可顺序访问<br/>支持批量操作 | BitTable<br/>HBase                   |

## 2.1 哈希分布

- 节点取余，一致性哈希，虚拟槽分区
- 数据漂移量大，不好做伸缩

# 四. 集群安装
## 1. 原生安装

```txt
127.0.0.1:7000  -- master
127.0.0.1:7001  -- master
127.0.0.1:7002  -- master
127.0.0.1:7003  -- replicate
127.0.0.1:7004  -- replicate
127.0.0.1:7005  -- replicate
```
**1. 配置节点：** 

> 最简集群配置，如： `vim 7000.conf`
```conf
port 7000
daemonize yes
dir "/usr/local/redis/data"
logfile "7000.log"
dbfilename "dump-7000.rdb"
cluster-enabled yes
cluster-config-file nodes-7000.conf
cluster-require-full-coverage no
```
```shell
# 分别修改相应的配置
sed "s/7000/7001/g" ./7000.conf > 7001.conf
sed "s/7000/7002/g" ./7000.conf > 7002.conf
sed "s/7000/7003/g" ./7000.conf > 7003.conf
sed "s/7000/7004/g" ./7000.conf > 7004.conf
sed "s/7000/7005/g" ./7000.conf > 7005.conf
```
**2. 启动服务：**
```shell
redis-server ./7000.conf
redis-server ./7001.conf
redis-server ./7002.conf
redis-server ./7003.conf
redis-server ./7004.conf
redis-server ./7005.conf
```
> 查看服务：`ps -ef | grep redis`

**3. 节点握手：**
```shell
redis-cli -p 7000 cluster meet 127.0.0.1 7001
redis-cli -p 7000 cluster meet 127.0.0.1 7002
redis-cli -p 7000 cluster meet 127.0.0.1 7003
redis-cli -p 7000 cluster meet 127.0.0.1 7004
redis-cli -p 7000 cluster meet 127.0.0.1 7005
```
> 测试一下：
> redis-cli -p 7000 cluster info    
> redis-cli -p 7000 cluster nodes    # 跟node-7000.conf一致
> redis-cli -p 7000 cluster set hello "world"  # 集群模式不可写入


**4. 分配槽**
创建脚本：`touch addslots.sh`
```shell
start=$1
end=$2
port=$3
for slot in `seq ${start} ${end}`
do
  echo "slot: ${slot}"
 redis-cli -p ${port} cluster addslots ${slot}
done
```
```shell
sh addslots.sh    0   5461 7000
sh addslots.sh  5462 10922 7001
sh addslots.sh 10923 16383 7002
```
**5. 测试**
```shell
# 集群模式存取
redis-cli -p 7000 -c set hello world
redis-cli -p 7003 -c get hello

# 单机模式存取
redis-cli -p 7000  get hello
redis-cli -p 7005  get hello
```
