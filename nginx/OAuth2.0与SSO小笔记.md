# OAuth2.0 & SSO 小笔记

## 1. OAuth 2.0
- **概念**：一种**授权协议**，让第三方应用在**不拿到用户密码**的前提下，安全获取用户信息和权限。
- **大白话**：给应用发一张**临时通行证**，不用交钥匙。
- **核心作用**：**授权、安全、可控**。
- **典型场景**：微信登录、QQ登录、支付宝登录。



<br/>



## 2. SSO（单点登录）
- **概念**：一种**登录机制**，用户**只登录一次**，就能访问所有互信系统，无需重复登录。
- **大白话**：一处登录，处处通行。
- **核心作用**：**共享登录状态、提升体验**。
- **典型场景**：公司内网、OA、邮箱、钉钉/飞书多应用免登。



<br/>



## 3. 核心区别（一句话）
- **OAuth2.0：管授权**——不给密码，只给权限。
- **SSO：管登录**——一次登录，全部免登。



<br/>



## 4. 关系
**实际中经常一起用：**
用 OAuth2.0 做安全授权，用 SSO 实现单点登录。
